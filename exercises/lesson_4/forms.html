Exercises: Forms

Exercise 1

Create a contact form that collects the user's first name, last name, email address, and phone number. Be sure to use appropriate input types, and prevent iOS devices from modifying the email input. See the MDN documentation for a hint.

Exercise 2

Add a select box to the form for phone type, with options of home, business, and mobile. Make sure the home option is pre-selected.

Exercise 3

Create a form for filtering news articles by a single category and any number of selected tags. Use radios for the category and checkboxes for tags. Make sure your form method is get to ensure the user can save and share the filtered URL.

Exercise 4

Convert the radios and checkboxes in the filter form to selects. Make sure you retain the ability to select any number of tags.

Exercise 5

Go over the markup for the previous number. There is a missing attribute with the select elements. What is it and what error does it raise with the w3c validator? Note that you have to complete the html structure before running it with the w3c validator.

Note: the issue being referred to in the exercise is a missing 'id' attribute on the 'select' elements; this is not applicable to my mark-up as I have nested the select elemements within the label elements.

Exercise 6

Provide the user with a way to reset the filter form.
